# x团队多智能体协同提示词

该仓库提供一套完全中文的系统提示词，用于搭建由“分析与规划”“执行与处理”“反思”“记忆”四个Agent组成的 x 团队。团队的目标是在接收到用户需求后，高效完成需求分析、任务拆解、执行落地、结果复核以及经验沉淀。

## 场景设定
- x 团队以项目团队的方式运作，每个 Agent 拥有清晰的职责边界和沟通接口。
- 分析与规划 Agent 需要根据用户输入和历史记忆，产出带有任务类型与目标的任务列表。任务类型仅支持“流程发起”“流程审批”“数据分析与报告呈现”“业务分析与报告呈现”。
- 执行与处理 Agent 作为类框架，先遵循基础提示，再加载动态执行模块（如流程发起、数据分析等）以选择适配的岗位身份并输出可验证的成果。
- 反思 Agent 负责校验执行结果与计划是否一致，并提出改进或复盘建议。
- 记忆 Agent 负责在整个流程中记录关键信息、经验与决策，供后续查询和复用。

## 统一任务结构
为了让任务在“规划—执行—反思—记忆”全流程中保持一致性，每个任务都必须严格遵循以下字段与顺序：

1. **任务编号**：使用 T1、T2… 等唯一编号。
2. **任务名称**：简洁说明任务主题。
3. **任务类型**：仅可填写“流程发起”“流程审批”“数据分析与报告呈现”或“业务分析与报告呈现”。
4. **任务描述**：用一句话描述“以何种角色，采取什么行动，实现何种目标”的任务概况。
5. **任务目标**：列出可验证的达成标准，可多条。
6. **任务执行角色**：指向明确的岗位名称，便于查询岗位职责。
7. **关键输入与资源**：完成任务所需的资料、系统、工具或支持。
8. **前置依赖与协作对象**：列出依赖的任务、干系人或必须完成的确认动作。
9. **预期产出与验收标准**：说明交付物形式及验收方式、成功阈值。
10. **关键风险与应对**：指出潜在阻碍及对应的预防或缓解措施。
11. **需要记忆的要点**：提示记忆 Agent 后续需要沉淀的知识或数据。
12. **需要反思的关注点**：标注反思 Agent 在复盘时要重点核查的事项。

所有 Agent 在读取、补充或回传任务信息时，都必须保留上述字段名称与顺序，确保结构完全一致。

## 目录结构
- `prompts/分析与规划智能体提示词.md`
- `prompts/执行与处理智能体提示词.md`
- `prompts/反思智能体提示词.md`
- `prompts/记忆智能体提示词.md`

## 使用指引
1. 在多智能体框架中，将对应文件的内容作为系统提示词注入相应 Agent，保持输出结构不变。
2. 若需扩展领域知识，可在不改变角色分工与任务类型约束的前提下追加补充说明。
3. 结合记忆 Agent 的积累以及反思 Agent 的反馈，持续迭代提示词外的业务知识，以增强团队整体表现。
4. 无论处于规划、执行、记忆或反思阶段，涉及任务的内容都必须沿用统一任务结构的 12 个字段及顺序，如需新增补充信息，应在不破坏原字段的前提下附加描述。
